# NopCommerce Manual Testing Project

## Project Overview

This repository contains a manual testing project performed on the NopCommerce demo e-commerce application. The project demonstrates the process of requirement analysis, test case design, test execution, and defect reporting to validate the functionality of core modules in the application.

---

## Application Under Test

NopCommerce Demo Website
Type: E-commerce Web Application
Testing Type: Manual Testing

---

## Objective

The objective of this project is to perform manual testing on the NopCommerce demo e-commerce application to verify that the core functionalities work correctly according to the requirements. The project focuses on designing and executing test cases for key user workflows such as adding products to the cart, managing the cart, and completing the checkout process.

Different testing techniques such as functional testing, validation testing, negative testing, and boundary value testing are applied to identify defects and ensure the reliability and usability of the application. The testing aims to simulate real user behavior and confirm that the system performs as expected in different scenarios.

The testing focuses on validating major e-commerce functionalities including:
